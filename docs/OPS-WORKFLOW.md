# 即梦内容工厂 — 运营工作流说明

---

## 一、每日工作节奏

```
下班前（准备期）→ 夜间（批量生产）→ 次日上班（验收上架）
```

---

## 二、下班前：准备物料

### 2.1 准备产品图

每个产品需要准备 **1~3 张**：
- **上身图**：模特穿戴效果图（白底/纯色背景最佳）
- **白底平铺图**：商品单独白底图（备用参考）

> 图片格式：JPG / PNG，分辨率建议 ≥ 720p

### 2.2 准备脚本变体

每个产品准备 **3~10 条** 不同的视频脚本（prompt），用于生成多样化内容。

**方法：** 把你的基础脚本扔给 ChatGPT / Gemini，让它输出 N 个变体

示例指令：
```
以下是一条 TikTok 视频脚本，请生成 5 个风格不同的变体，保持模板格式，
保持同类目（泰国 TikTok Shop 女装），语言保持泰语：
[原始脚本]
```

---

## 三、提交任务

### 3.1 打开多空间浏览器

先在多空间浏览器里把所有即梦账号打开，确认已登录。

> 不需要手动操作即梦页面，工厂会自动接管

### 3.2 打开管理后台

双击 `start.bat` → 浏览器自动跳转到 `http://localhost:8001`

### 3.3 创建产品

进入 **「产品」** 页面 → 「新建产品」：
- 填写产品名称（建议用 SKU 编号，方便后续归档）
- 上传产品图

### 3.4 批量提交任务

进入 **「提交」** 页面：
- 选择产品
- 粘贴脚本变体（每条一个任务）
- 选择视频时长（4~15 秒，建议 8~10 秒）
- 点击「批量提交」

> 系统自动分配账号、排队生产，无需手动干预

---

## 四、夜间自动运行

提交后关窗口即可，不用管。系统在后台：

1. 自动调度账号，保持每账号 10 个并发任务
2. 自动检测完成状态，下载视频
3. 自动去重（同一视频 URL 不会重复下载）
4. 视频保存到 `outputs/<产品名>/` 目录

---

## 五、次日上班：验收

### 5.1 查看任务状态

打开后台 **「任务」** 页面：

| 状态 | 含义 |
|------|------|
| 🟡 Pending | 排队中 |
| 🔵 Generating | 生成中 |
| 🟢 Done | 已下载 |
| 🔴 Failed | 失败（可重提） |

### 5.2 取视频文件

进入 `outputs/<产品名>/` 文件夹，视频已按产品分类存放。

### 5.3 处理失败任务

失败任务在「任务」页面点「重试」即可重新加入队列。

---

## 六、注意事项

| 事项 | 说明 |
|------|------|
| 多空间浏览器必须保持开启 | 工厂通过 CDP 连接已登录的账号，浏览器关闭则无法生产 |
| 提交前确认账号已发现 | 首页/账号页确认账号列表非空，否则点「重新发现账号」 |
| 不要手动操作即梦页面 | 工厂自动化期间手动操作可能导致任务冲突 |
| 视频时长 4~15 秒 | 超出范围系统会忽略，使用默认 10 秒 |
| 不指定模型 | 系统固定使用 Seedance 2.0 Fast，无需选择 |
