# CLAUDE.md - 即梦内容工厂

## 项目概述
即梦内容工厂：基于即梦（Jimeng）的自动化视频生成系统，面向 TikTok Shop 泰区/印尼区短视频内容生产。

## 执行流程

本项目通过 orchestrator-harness 执行任务。**不要直接执行高风险或需要验证的操作**，而是通过 harness 走完整的执行→验证→决策流程。

### 工作流

1. **规划阶段**：与用户讨论方案方向，写好代码
2. **执行阶段**：通过 harness 提交任务，Codex 执行并审查方案
3. **验证阶段**：harness 自动验证执行结果
4. **决策阶段**：
   - 通过 → 自动完成，飞书群收到「✅ 任务完成」
   - 可重试 → 自动重试，飞书群收到「⚠️ 重试中」
   - 需人工 → 飞书群收到「🚨 需决策」，等待用户在飞书回复后继续

### 调用方式

当需要提交任务执行时，运行：

```bash
cd E:/orchestrator-harness && source .env && npx tsx scripts/jimeng-run.ts "<需求描述>" tred
```

harness 会自动完成：Codex 执行 → 验证 → 飞书通知 → 等待决策（如需要）→ 返回结果。

### 飞书群
- 群名：Claude 即梦工厂
- 所有任务状态变更自动推送到该群
- 需要人工决策时，用户在群里**引用回复**卡片消息即可（接受/拒绝/重试/任意指示）

### 配置
- Harness 路径：`E:\orchestrator-harness`
- Runs 目录：`E:\即梦内容工厂\runs\`
- Runner：Codex 真实执行
- 超时：10 分钟无回复自动 abort
